# Contributing to One Mandaluyong Super App

Full contribution guidance lives in **Section 30 (Contributing)** of [README.md](./README.md), including who can contribute, the contribution process, and expectations.

Quick summary:
1. Open a tracked issue before starting work.
2. Request architecture review for changes affecting shared contracts, security, identity, or data governance.
3. Follow the Branching Strategy (Section 24) and Coding Standards (Section 23).
4. Include tests appropriate to the change's risk level.
5. Open a pull request using the provided template — peer review is mandatory for every change.

Security issues follow the incident reporting process in Section 18 (Security Architecture), not this general process.
